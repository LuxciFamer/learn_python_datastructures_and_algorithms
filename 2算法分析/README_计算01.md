# 计算01：线性时间求第 k 小元素

`计算01.py` 实现了两种方法，在无序列表中求第 `k` 小元素（`k` 从 1 开始）：

- `select_kth`：BFPRT（Median of Medians），最坏 `O(n)`
- `select_kth_by_sort`：排序法，`O(n log n)`

## 复杂度对比

- `select_kth`（BFPRT）：最坏时间 `O(n)`，额外空间 `O(n)`（当前实现用新列表做三路划分）
- `select_kth_by_sort`（排序法）：时间 `O(n log n)`，额外空间取决于排序实现

## 为什么是线性阶

BFPRT 每轮：

1. 把元素按 5 个一组分组，组内取中位数（总计 `O(n)`）。
2. 递归求这些中位数的中位数作为枢轴（规模约 `n/5`）。
3. 用该枢轴做划分后，递归只进入一侧，且该侧规模最多约 `7n/10`。

因此有递推式：

`T(n) = T(n/5) + T(7n/10) + O(n)`

该递推解为 `O(n)`，所以算法是线性阶（最坏情况也成立）。
